
  # Echo Flo Prototype

  This is a code bundle for Echo Flo Prototype. The original project is available to see at https://ratio-find-89492101.figma.site

  ## Running the code

  Run `npm i` to install the dependencies.

  Run `npm run dev` to start the development server.

## ✨ Inspiration

We were inspired by the concept of Interceptive Resonance. The silent, biological conversation happening in a room that often feels invisible to people with social anxiety or neurodivergence. 🧠💭

As first-generation students, networking events can feel especially overwhelming because imposter syndrome is so common. That pressure can lead to social anxiety and can negatively affect mental health. 💙

We wanted to build a tool that gives people a kind of “sixth sense” that transforms overwhelming emotional noise into a clear, navigable map. 🗺️🔮

## 🚀 What It Does

EchoFlo is a relational data engine that treats a room full of people like a live graph database. Using AR and haptics, it:

Visualizes the invisible 👁️
Maps individuals and gives them a color aura influenced by their emotions and facial expressions.

Reveals the room’s overall vibe 🌐
Helps users understand the energy of a space and see how people are interacting.

## 🛠️ How We Built It

We architected the system by first prototyping different designs, with careful attention to color choice. 🎨 We wanted the interface to feel calming and accessible, so we avoided harsh colors and focused on tones that were easy on the eyes.

We iterated on the design about four times before moving into Figma Make. Throughout the process, we refined both the visual language and the user experience.

EchoFlo includes an app interface with:

Home 🏠

Analytics 📊

Settings ⚙️

Biometric Health Dashboard ❤️

The system is designed to work alongside Echo Lenses, a companion wearable tool. 👓 Once the glasses are on, users can view the emotions and vibes of a room in real time, along with the people around them. The Live and Scene views represent what users would experience through the lenses.

## ⚠️ Challenges We Ran Into

Our biggest hurdle was data overload. 📉

At first, we realized that showing everything like heart rate, breathing rate, and other biometric signals could actually make social anxiety worse instead of helping. Because of that, we had to rethink how much information should be displayed.

This led us to iterate on our Sensory Control settings, where we introduced a vibe opacity slider 🎚️ so users could dim the data and focus more on human connection rather than overwhelming metrics.

## 🏆 Accomplishments We’re Proud Of

We are especially proud of creating our logo and learning how to use Figma for the first time for some of us. ✍️✨

What felt most meaningful was moving beyond simply identifying a problem and actually building a solution through Figma Make. Seeing everyone learn, collaborate, and turn this idea into something that feels like a real medical and wellness aid was incredibly rewarding. 🤝💡

## 📚 What We Learned

We learned about LiDAR, and we embraced the idea that “Clear is Kind.” 💬

For someone with ADHD, knowing that a room is “High Friction” can be much more helpful than being told it is simply “Tense.” That clarity can make social environments easier to understand and navigate.

We also discovered the power of Haptic Sonification 🎵📳 which is the idea that you do not always need to see the data in order to feel the safety and connection of a synchronized group.
